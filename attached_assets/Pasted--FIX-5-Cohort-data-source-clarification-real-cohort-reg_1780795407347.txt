FIX 6 — Perpetual spinner with no timeout on eligibility and aggregate
════════════════════════════════════════

Both eligibility check and aggregate query can spin forever.

For BOTH flows, add this pattern after submitting the transaction:

const TIMEOUT_MS = 120_000 // 2 minutes

const result = await Promise.race([
  waitForResult(txHash),
  new Promise((_, reject) => 
    setTimeout(() => reject(new Error('timeout')), TIMEOUT_MS)
  )
])

If timeout fires, show:
"⏱️ The FHE computation is taking longer than expected.
Your transaction was submitted: [Etherscan link]
The result may still arrive on-chain — check back in a few minutes.
You can also verify directly on Etherscan."

Always show the Etherscan link regardless of timeout.
